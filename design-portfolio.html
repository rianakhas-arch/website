CTMFs

Methods, tools, and frameworks organized around purpose, evidence, and reflection.

Use this page as a reference handbook. Each CTMF entry should explain what the method is for, when you used it, what evidence supports it, and what you learned about its usefulness in engineering design practice.

How to use this page

  • Keep only CTMFs you can connect to real project evidence
  • Write one or two clear paragraphs per entry rather than long notes
  • Reuse the same field pattern so the page stays easy to scan

Field Pattern

Each card below uses the same four placeholder fields.

Purpose

Paste what the CTMF is meant to help you do in a design process.

When used

Paste the project phase, moment, or type of problem where you used it.

Evidence

Paste the artifact, document, sketch, prototype, or observation that proves it was used.

Reflection

Paste what the CTMF helped you notice, clarify, or decide, including any limitations.
